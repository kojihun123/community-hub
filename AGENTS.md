# AGENTS.md

## 프로젝트 개요

이 프로젝트는 포트폴리오와 취업 준비를 목적으로 만드는 Laravel 기반 커뮤니티 플랫폼이다.
핵심 목표는 기능 수를 무작정 늘리는 것이 아니라, Laravel 기본기와 실제 커뮤니티 서비스 설계 능력을 보여주는 것이다.

## 핵심 스택

- Laravel
- Blade
- Alpine.js
- Tailwind CSS
- MySQL
- Redis

필요 시 아래 도구를 추가로 도입할 수 있다.

- Laravel Breeze
- Laravel Sanctum
- Laravel Reverb
- Laravel Echo
- PHPUnit

## 작업 원칙

- 불필요한 추상화보다 단순하고 유지보수하기 쉬운 Laravel 구조를 우선한다.
- 과하게 큰 기능보다 끝까지 완성할 수 있는 구현을 우선한다.
- 기본 렌더링 방식은 Blade 서버 렌더링으로 유지한다.
- Alpine.js는 가벼운 UI 인터랙션에만 사용한다.
- 관리자 화면은 무거운 관리자 패키지 대신 직접 구현하는 방향을 우선한다.
- 작업은 `docs/DEVELOPMENT_PLAN.md`의 단계별 계획을 따른다.

## 우선 참고 문서

구조나 기능 방향을 결정하기 전에 아래 문서를 먼저 확인한다.

- `docs/PLAN.md`
- `docs/DEVELOPMENT_PLAN.md`
- `docs/ENVIRONMENT_SETUP.md`

구현 내용과 문서가 충돌하면, 먼저 문서 기준 의도를 확인한 뒤 작업한다.

## 아키텍처 선호 사항

- 컨트롤러는 가능한 한 얇게 유지한다.
- 비즈니스 로직이 커지면 서비스 클래스로 분리한다.
- 검증은 필요 시 Form Request로 분리한다.
- Eloquent 관계는 명확하게 정의하고, 불필요하게 복잡한 쿼리는 피한다.
- 반복되는 UI는 Blade 컴포넌트로 정리한다.

## 기능 우선순위

특별한 이유가 없다면 아래 순서로 구현한다.

1. 인증과 기본 레이아웃
2. 게시판, 게시글, 댓글
3. 관리자 운영 기본 기능
4. 좋아요, 북마크, 신고 같은 커뮤니티 상호작용
5. 제재, 알림, 탈퇴 정책 같은 운영 기능
6. API, Redis 캐시, 브로드캐스팅, 테스트 같은 고급 기능

## 기본적으로 피할 것

- 명시적 요청이 없으면 Vue, React, Inertia를 도입하지 않는다.
- 초반부터 복잡한 추상화 구조를 만들지 않는다.
- 대형 서드파티 관리자 프레임워크를 기본 선택으로 쓰지 않는다.
- 문서에 없는 범위 확장을 함부로 하지 않는다.

## 문서 관리

- 기획 및 설계 문서는 `docs/` 아래에 둔다.
- 구현 방향이 크게 바뀌면 관련 문서를 함께 수정한다.
- 추측성 대규모 문서 수정보다, 작고 정확한 문서 갱신을 선호한다.
