# CommunityHub Development Plan

## Overview

이 문서는 [`PLAN.md`](/home/jhko/portfolio/PLAN.md)의 설계를 실제 구현 작업으로 옮기기 위한 개발 체크리스트다.
목표는 모든 기능을 한 번에 구현하는 것이 아니라, `핵심 기능부터 단계적으로 완성`하는 것이다.

기본 원칙은 다음과 같다.

- 먼저 동작하는 커뮤니티를 만든다.
- 이후 운영 기능과 고급 기능을 붙인다.
- 마지막에 API, 실시간, 캐시, 테스트를 보강한다.

## Recommended Build Order

실제 개발은 아래 순서로 진행하는 것을 권장한다.

1. 개발환경과 인증 기반을 먼저 고정한다.
2. 게시판, 게시글, 댓글이 동작하는 최소 커뮤니티를 만든다.
3. 관리자 화면으로 핵심 리소스를 운영 가능하게 만든다.
4. 좋아요, 북마크, 신고 같은 커뮤니티 상호작용을 붙인다.
5. 제재, 숨김, 탈퇴 정책 같은 운영 기능을 완성한다.
6. 마지막에 API, 캐시, 이벤트, 실시간, 테스트를 보강한다.

## Suggested Implementation Order

1. 인증과 기본 레이아웃부터 만든다.
2. 게시판, 게시글, 댓글이 동작하는 최소 커뮤니티를 완성한다.
3. 관리자 화면으로 게시글/댓글/게시판을 관리할 수 있게 만든다.
4. 좋아요, 북마크, 신고를 붙여 커뮤니티 상호작용을 강화한다.
5. 제재 정책과 운영 이력을 붙여 서비스 운영 구조를 완성한다.
6. 마지막에 API, Redis, Event/Job, Reverb/Echo, PHPUnit을 보강한다.

## Done Criteria

- 회원이 가입하고 글과 댓글을 작성할 수 있다.
- 게시판 중심 커뮤니티 흐름이 완성되어 있다.
- 관리자가 신고, 게시글, 댓글, 공지를 운영할 수 있다.
- 제재와 숨김 처리에 사유와 이력이 남는다.
- Laravel 중급 기능이 실제 코드에 최소 한 번 이상 반영되어 있다.
